## What it does

`/api/metrics` exposes the application's Prometheus metrics in text format for
scraping. It serves the custom prom-client `Registry`
([metrics.ts:7](../../apps/site/src/lib/observability/metrics.ts#L7)) and
protects production scrapes with a bearer token, while staying open for local
development. It is the metrics half of the
[observability architecture](../concepts/observability-architecture.md).

## How it is configured

The token is resolved from one of two sources, in order: an explicit
`METRICS_BEARER_TOKEN` env var, or an SSM SecureString at the path in
`METRICS_TOKEN_SSM_PATH`
([metrics/route.ts:12-13,42-55](../../apps/site/src/app/api/metrics/route.ts#L42-L55)).
SSM lookups are cached for 5 minutes (`TOKEN_CACHE_TTL_MS`) to avoid an API call
per scrape, time out after 3 seconds, and back off for 60 seconds after a
failure
([metrics/route.ts:16-27](../../apps/site/src/app/api/metrics/route.ts#L16-L27)).
The route pins the Node.js runtime and is non-cached
([metrics/route.ts:195-197](../../apps/site/src/app/api/metrics/route.ts#L195-L197)).

## How it integrates with the rest of the system

Prometheus scrapes this endpoint every 15-30 seconds with an
`Authorization: Bearer <token>` header. The token is generated by the monitoring
EC2 at boot and stored as a SecureString
([metrics/route.ts:34-37](../../apps/site/src/app/api/metrics/route.ts#L34-L37)).
Token comparison is constant-time via `timingSafeEqual`, with an explicit length
check first
([metrics/route.ts:119-128](../../apps/site/src/app/api/metrics/route.ts#L119-L128)).

## Failure modes

The endpoint is designed to **fail closed in production**: if no token source is
configured there, it returns `503`
([metrics/route.ts:47-55,150-156](../../apps/site/src/app/api/metrics/route.ts#L150-L156));
a bad/missing bearer returns `401`
([metrics/route.ts:158-166](../../apps/site/src/app/api/metrics/route.ts#L158-L166)).
In local/test (no SSM path, not production) auth is disabled and metrics serve
openly. On an internal error it still emits a minimal `nextjs_up 0` body with
status 500 so a scrape failure is itself observable
([metrics/route.ts:181-186](../../apps/site/src/app/api/metrics/route.ts#L181-L186)).

## Operational notes

prom-client requires Node-only APIs and must not be bundled or run on the Edge
runtime — hence `export const runtime = 'nodejs'` and its inclusion in
`serverExternalPackages`. Getting this wrong produces build warnings and a
broken registry; see the companion troubleshooting doc.
